1. Portée Technique du Projet

DOCU243 est une plateforme gouvernementale transactionnelle combinant :

Identité numérique hybride

Catalogue de services administratifs

Paiement Mobile Money

Traçabilité complète des opérations

Inclusion offline / USSD

Le système doit être :

Modulaire

Scalable

API-first

Offline-first

Sécurisé by design

2. Environnement de Développement (Replit)
2.1 Contraintes et choix Replit

Replit sert pour :

Prototypage rapide

Démonstration fonctionnelle

Tests utilisateurs

API backend MVP

⚠️ Limites connues :

Pas d’USSD natif → simulation via API

Pas de stockage souverain → abstraction nécessaire

Pas de HSM → PKI simulée

👉 Conclusion : Architecture portable vers cloud souverain.
